The Nymi Band

The Nymi band provides individuals with persistent identity via authentication using a person's unique ECG. It comes with a companion application, the Nymi Companion App (NCA). The NCA is utilized by users to establish their identity on first use (enrollment) and for authentication on subsequent uses. Once the a wearer is authenticated, they remain authenticated as long as the device is worn by the same person.

The wearer can then identify themselves with Nymi Enabled Applications (NEAs) without the need for other credentials such as passwords.

Communication between the Nymi band and either the NCA or NEAs occurs through Bluetooth Low Engergy (BLE). On Windows based machines, an external BLE dongle is required, along with the correct drivers and the Nymi Bluetooth Service. A complementary dongle (BLED112, Bluegiga) is provided with each Nymi.




How the Nymi Band Identifies its Wearer

Enrollment and Authentication

The Nymi band identifies wearers based on their electrocardiogram (ECG). When a user puts on their Nymi for the first time, a comprehensive ECG profile is created and saved, using the NCA. This process is named enrollment. The NCA is currently supported on Windows and OSX platforms.

The next time that the same user puts on the same Nymi band, ECG is captured and matched to the stored profile. This process is named authentication. Provided the newly captured ECG matches the ECG during enrollment, the Nymi band enters an authenticated state and becomes active. It remains in this state until it is removed from the wearer's wrist (either the band is cut, or the clasp is opened).

An unauthenticated Nymi cannot be used with NEAs and encrypts its internal data until it is successfully authenticated.

This process is handled by the NCA, which we provide.

Enabling NEAs

Nymi Enabled Applications (NEAs) can be allowed to work with a particular Nymi band by the user. This process is known as provisioning.

A provision is a bridge between a NEA and a Nymi. After the bridge is established, the application can detect and communicate with the same Nymi in the future as long as that Nymi band is authenticated. Once a provision is established, it persists until the Nymi undergoes factory reset, or when the user opts out of the app using the Nymi Companion App. NEAs can only communicate with a Nymi band if it is authenticated.

An NEA can look for a previously provisioned Nymi through a process called validation. The validation process as a whole consists of first finding and then validating the Nymi Band. It allows the capability to determine the user profile that the Nymi Band was originally provisioned for and for using the Nymi band for application specific needs such as vibrating the Nymi band, using it to generate keys, etc.

It is a two step process depending on your application's use case. The first step is mandatory and requires your application to find the Nymi Band. This finding will result in a strong or weak boolean indicator to show if the original Nymi Band that was authorized by the NEA has been found. The strong boolean indicator is a quick way to confirm the Nymi Bandâ€™s identity. If a weak boolean indicator is received you will need to proceed to the second step.

The second step involves the nclValidate API callback. If your application needs the Nymi Band for more than determining the user profile (for example, to generate keys or to make the Nymi Band vibrate), your application will need to proceed to placing the nclValidate API callback which provides greater security and opens the communication channel to the Nymi Band.

Once the Nymi has been validated, application-specific behaviour can occur and data can be collected from the Nymi through the Nymi Communications Library (NCL) API calls. The only exception is nclGlobalSign, which does not require the Nymi to be validated. Developers must take care of both the provision and validation through a sequence of API calls. The API is provided via the Nymi Communications Library (NCL).

  • NEAs on subsequent usage validate against the provisioned Nymi band. Once validation is successful, the NEA can assume an authenticated user.

The developer version of the Nymi bands can only be provisioned up to 6 times with NEA (Nymi Enabled Applications). To provision more than 6 times, you will need to delete the existing profile(s) for the band from the NCA (Nymi Companion App). This will enable the developer version of the Nymi band for authorization once more..




Capabilities of the Nymi

The band provides a number of other other capabilities in addition to the the basic authentication. These capabilities are either security related or data generated by the Nymi band that can be used by the NEA.

The security related features allow NEAs to perform signature functions, general key pairs and pseudorandom numbers. These topics are covered more in the Security Tutorial.

The band can proivde the following data for use by an NEA:

  • Proximity through RSSI reading
  • Electrocardiogram (ECG)

The Streams section of the Nymi Communication Library Reference contains functions for reading ECG data. nclGetRssi queries the Nymi for RSSI values.

Notifications to the User

A Nymi can provide feedback to the wearer in two ways:

  • Visual: there are 5 LED lights located on the side of the central housing
  • Haptic: there is a vibration motor located in the wristband

The LED lights and the vibration motor are primarily used for built-in operations such as reporting the state of the Nymi. However, Nymi-Enabled Applications can use the haptic motor for simple notifications through nclNotify.
