iOS with Xcode

To see a video tutorial on getting started with Nymi Band development on the iOS platform, see Getting started.

The iOS implementation uses CoreBluetooth. This is Apple's native BLE communication stack and was available as of iOS 6. More information regarding their stack can be found here

The SDK provides two different libraries for the NCL;

  • libNCLiOS.a uses CoreBluetooth to communicate with the Nymi Band
  • libNCLNetiOS.a uses TCP/IP to communicate with a Nymulator
  1. Get ncl.h and the appropriate libNCL library, provided in the Mac SDK, and place them in your project directory. They are located in the NCL folder.
    • If you wish to communicate with the Nymi Band, use libNCLiOS.a
    • If you wish to communicate with the Nymulator, use libNCLNetiOS.a
  2. Open Xcode and create a new iOS project if you don't have one
  3. In the Build Settings tab of the project, search for Heder Search Paths and add the directory for ncl.h. If the ncl.h is beside the project file, you can add "." to include the current directory.
  4. Drag ncl.h and the appropriate libNCL library into your project. This will add the library with an absolute path. To add a relative path, you can click on the file and on the Utility panel (right panel) you can change the Location to Relative to Project
  5. In the Build Phases tab of the project click Copy Files
    • For the Destination field, select Executables
    • Clear the subpath
    • Unselect "Copy only when installing"
    • Click the '+' button and dd the libNCL dylib
  6. In the Build Settings tab of the project, search for Runpath Search Paths and add "@executable_path/" (the product executable directory)
  7. The following code snippet, written in C++, can be included if using libNCLNet.dylib
    • Ensure the Nymulator is running before building and running the code
      #include "ncl.h" //Includes the ncl library to your project
      ...
      /*
      Function for handling the events thrown by the NCL
      @param[in] event NclEvent that contains the event type and member variables
      @param[in] userData Data that needs to be passed to the callback functions if provided
      */
      void callback(NclEvent event, void* userData){
      if(event.type == NCL_EVENT_INIT){
      if(event.init.success){
      //can start using NCL functions
      }else{
      //gracefully handle error
      }
      }
      }
      ...
      void initNCL(){
      //Only if using the Nymulator. This function is required to direct the communication
      //to the computer hosting the Nymulator.
      //127.0.0.1 is assuming the Nymulator is running on localhost
      //9089 is the port the Nymulator is listening on
      if(!nclSetIpAndPort("127.0.0.1", 9089)){
      //gracefully handle error
      }
      //Initializes the Nymi Communication Library
      //'callback' refers to the function that will be handling the NCL callbacks
      //NULL indicates there is no data to be passed to the NCL callbacks
      //'NEAName' is the name of this NEA program that will be provisioned in the Nymi
      //NCL_MODE_DEFAULT is set to configure the initialization to communicate with a default Nymi/Nymiulator
      //stderr refers to the stream where the NCL logs will be printed
      if(!nclInit(callback, NULL, "NEAName", NCL_MODE_DEFAULT, stderr)){
      //gracefully handle error
      }
      }
  • If the NCL was successfully added to your project and it is communicating with the Nymulator, you will be able to see "An NEA has attached"
  • If using libNCL.dylib, the nclInit function must be initialized with NCL_MODE_DEFAULT instead of NCL_MODE_DEV. Additionally, nclSetIpAndPort can be omitted
