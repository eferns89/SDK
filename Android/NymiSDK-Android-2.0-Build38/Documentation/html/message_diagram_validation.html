Validation Message Diagram

The validation process is the main process that is used repeatedly after provisioning, to securely recognize the provisioned Nymi. This process is divided into several steps;

  • Find: By providing a list of provisions to nclStartFinding, this function will return an NCL_EVENT_FIND whenever an active provisioned Nymi is found. This is a passive process that does not establish a connection with the Nymi
    void callbackFunction(NclEvent e, void* userdata){
    if (e.type == NCL_EVENT_FIND){
    //connect to found device or collect multiple find events
    //and apply a heuristic to connect based on proximity (RSSI)
    }
    }
    }
    int findFunction(NclProvision* provision, int numOfProv){
    ...
    NclBool res = nclFind(provision, numOfProv, NCL_FALSE);
    ...
    }
  • Validate: If a Nymi is found, nclValidate estbalishes a connection and performs a handshaking protocol to first guarantee the authenticity of the connected Nymi, and then establish a secure session for data query.
    void callbackFunction(NclEvent e, void* userdata){
    if (e.type == NCL_EVENT_VALIDATION){
    //A trusted secure session is now established
    }
    }
    int validationFunction(int nymiHandle){
    ...
    NclBool res = nclValidate(nymiHandle);
    ...
    }

The validation process alone is sufficent to prove the identity of the Nymi, and subsequently the wearer. If additional information is required, such as global signature key pair request, or symmetric keys request, this can only be performed once NCL_EVENT_VALIDATION is received.
