Nymulator

The Nymulator is a stand-alone application that simulates an active Nymi. It is currently available for both the Windows and OS X platform and can be found in the Windows and Mac OSX SDK respectively.

The current Nymulator does not simulate the sensor streaming (i.e. Accelerometer, Gyroscope or ECG), which is available with the Nymi Band. There is also a limit of a single NEA provision per Nymi on the Nymulator. A new provision will override the exisiting provision.

Programming with the Nymulator requires a different flavour of the NCL. The communication between the NEA and Nymulator is done through TCP/IP as opposed to Bluetooth Low Energy. Subsequently, the host machine for the Nymulator can be different from the NEA as long as they are on the same network. (e.g. Mac OS X NEA to Windows Nymulator, Windows NEA to Windows Nymulator, iPhone hardware to Windows Nymulator, iPhone simulator to Windows Nymulator etc.). In such cases, you need to specify the location of the device hosting the Nymulator by providing the IP address of the device by using nclSetIpAndPort.

1 Note:
2 - On a Windows computer, the Nymulator cannot be running at the same time as the Nymi Bluetooth Service. It also requires the Microsoft Visual Studio 2013 Redistributable to run.
3 - The port of the Nymulator is always set to 9089.

The Nymulator Interface

The Nymulator is separated into 4 main panels;

Screenshot of the Nymulator

Interacting with the Nymulator

The left panel is used to manage the simulated Nymis;

  • New: Creates a new Nymi instance and adds it to the Nymi List. Each Nymi can contain a different list of provisioned NEAs, generated symmetric keys, and generated local and global signing keys.
  • Load: Load a list of previously saved Nymis, their list of provisioned NEAs and their generated keys.
  • Save: Save the current list of Nymis, provisions and keys.

Currently, the Nymulator supports only one active Nymi at a time. To set a particular Nymi to be active, click on the deisred Nymi from the Nymi List, and click the right arrow. This will highlight the active Nymi with a red outline. A Nymi can later be deleted by clicking the minus symbol button.

The middle panel is used to reflect the state of the Nymi;

  • The top image reflects the state of the Nymi. LEDs will be visible when the Nymi is required to show the Agreement pattern, or in provisioning mode.
  • The RSSI Value is a scroll bar that allows you to simulate the RSSI of the active Nymi; the lower the value the further the Nymi is from your device, and simiarily, the higher the value, the closer the Nymi. The current range is from -90 to 10dB

The right panel is used to interact with the Nymi;

  • Provision: Places the active Nymi into provisioning mode. By clicking this button, you should see the Nymi image display an LED pattern for the duration of the provisioning mode.
  • Disconnect: Disconnects the current connected NEA
  • Nymi Log: Text field that logs the requests and responses from the Nymi.

The bottom panel is used to display the NEA name that is currently connected to the Nymulator. This name is specified by the NEA when initializing the NCL with nclInit
