Android with Eclipse

To see a video tutorial on getting started with Nymi Band development on the Android platform, see Getting started.

The Android implementation of the NCL is a JNI interface that interacts with the native NCL code. It uses Android's native BLE communication stack, which was available as of Android 4.3 (API Level 18). More information regarding their stack can be found here .

To setup your Android project to use the NCL,

  1. Create your Eclipse Android project if you don't have one
  2. Open AndroidManifest.xml, and add the following permissions if they do not already exist:
    1 <uses-permission android:name="android.permission.BLUETOOTH" /> <!-- Needed for communicating with the Nymi Band -->
    2 <uses-permission android:name="android.permission.INTERNET" /> <!-- Needed for communicating with the Nymulator -->
  3. Under your Android project folder, create a new folder, libs
  4. In the Android SDK Android/libs folder of the downloaded SDK, copy NCL.jar into the newly created libs folder.
  5. Copy the architecture(s) you want to support to the libs folder. We currently support armeabi, armeabi-v7a, x86, and mips. We recommend that you should always include armeabi-v7a folder as it is widely used today. In addition, x86 is also recommended, especially when you want to use Android emulator with hardware acceleration.
  6. In your project, if you are programming for the Nymulator add the following snippet of code:
    import com.bionym.ncl.*;
    ...
    private void initNCL(){
    /*
    * Initializes the NCL to point to a Nymulator.
    * The ip is a string that is set to the address of the computer
    * running the Nymulator. If the NEA is being runned through the Android
    * simulator and running on the same computer as the Nymulator
    * the ip should be set to 10.0.2.2
    */
    NclHelper.initialize(this, ip, 9089);
    /*
    * Initializes the NCL library
    * This will initialize the NCL to have the events handled by cb, and
    * communicate with dev mode Nymis, and save all the logs to a text file
    */
    Ncl.init(cb, null, "neaName", NclMode.NCL_MODE_DEFAULT, Environment.getExternalStorageDirectory() + "/AB_MIRROR.txt");
    }
  7. Build and run your Android project. If successfully the NCL added to your project, you will see you "An NEA has attached"
  • If you are programming for the Nymi Band, the above snippet needs to change to the following
    import com.bionym.ncl.*;
    ...
    private void initNCL(){
    /*
    * Initializes the NCL use Android's Bluetooth Low Energy Library
    */
    NclHelper.initialize(this);
    /*
    * Initializes the NCL library
    * This will initialize the NCL to have the events handled by cb, and
    * communicate with Nymi Bands, and save all the logs to a text file
    */
    Ncl.init(cb, null, "neaName", NclMode.NCL_MODE_DEFAULT, Environment.getExternalStorageDirectory() + "/AB_MIRROR.txt");
    }
    An full Android project is available in the Examples folder of the SDK package. The AndroidBase application provides an example of provisioning and validating a Nymi.
